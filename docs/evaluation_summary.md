## 对比结果

| 配置 | Hit@5 | MRR@5 | 检索 P95 | 阈值 | OOD 误答率 |
|---|---:|---:|---:|---:|---:|
| Dense-only | 0.900 | 0.844 | 59.56 ms | 0.6526 | 10% |
| Dense + BM25 + RRF | 0.950 | 0.861 | 43.86 ms | 0.6526 | 10% |
| 完整精排：12 候选 / 512 | 0.950 | 0.919 | 7,777.50 ms | 0.7265 | 0% |
| 最终精排：5 候选 / 256（FAISS 知识库复跑） | 0.950 | 0.919 | 1,461.68 ms | 0.7256 | 0% |
| 实验精排：5 候选 / 192 | 0.950 | 0.908 | 2,101.31 ms | 0.7215 | 0% |

P95 来自不同进程的热缓存运行，Dense 和 Hybrid 的几十毫秒差异不应解释为 Hybrid 必然
更快。Cross-Encoder 延迟量级差异稳定且具有工程意义。

## 结论

1. BM25 + RRF 将 Hit@5 提升 5 个百分点，说明中文通知标题、年份和考试名称需要关键词
   通道，Hybrid 应保留。
2. Cross-Encoder 不再提高 Hit@5，但将正确来源提前，使 MRR 从 0.861 提升到 0.919；
   对生成上下文排序有价值。
3. 将精排输入从 12×512 降至 5×256，质量指标不变，P95 下降约 71%，因此采用该配置。
4. 继续缩到长度 192 只节省约 170 ms，却损失 MRR，因此不采用。
5. 独立 FAISS 知识库重构后热检索 P95 1.462 秒，刚好达到 1.5 秒目标，但余量很小；
   生产增强仍应尝试 ONNX/INT8，或为简单问题提供 Hybrid 快速模式。
6. 默认精排阈值固化为 0.7256。关闭精排后分数语义变化，必须同步使用 Hybrid 阈值
   0.6526，不能共用同一阈值。
7. 5 题真实生成样本的严格 `citation_gold_title_precision` 为 0.667。逐题检查显示，扣分项
   多为成绩复核通知、补选通知、学院复试细则等补充来源；这个指标只判断是否命中单一
   gold 标题，不等于事实—来源正确率。全量引用正确率仍需逐条人工或独立评审模型判断。
